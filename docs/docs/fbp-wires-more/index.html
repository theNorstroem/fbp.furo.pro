    Working with sub optimal components #

    Some components that you may want or have to use are not dispatching events. Therefore, furo FBP has some features to work with them too.

    Wireing responses from method calls #

    When the method that you have wired only returns data that you want to use, you can wire the response with @-ƒ-methodname.

    The response of the calculate(n) method is avaliable on the wire --calculatedSqrRoot.

    Spread arguments #

    When a receiver mehtod accepts multiple arguments or is a spread operator and the data on the wire is spreadable, furo FBP will handle this for you correctly.
